Modules

common.js

The common.js module is the main point of entry for all UI Buildkit modules that are going to use mvApi. It is the only module that is intended to expose public methods and by doing so it creates the public mvApi itself.

In order to use it, the following steps need to be taken in the module:

  • A module needs to require 'modules/mvapi/common' using RequireJS
  • A module needs to require set of inital models for the module, e.g. 'modules/mvapi-example/mv-models'
  • To display a component on a page, the 'mvApi.render(id)' method needs to be invoked, where 'id' is an ID of a model to be rendered - a corresponding initial data for this model should be already loaded in the previous step.
  • To change anything within a component rendered previously with mvApi, only a change to its model needs to be provided. To do so, the 'mvApi.render(id, modelChange, callback)' should be invoked.

    The 'id' is a REQUIRED ID of a model to be rendered

    The 'modelChange' is an OPTIONAL JSON object containing only those keys, which values need to be changed

    The 'callback' is an OPTIONAL method that should be invoked after the view has been rendered.

Please note the sequence of the optional params provided (model change or callback) is not mandatory. mvApi recognizes the type of paramater provided and uses them accordingly. So you can do either one of the following:

  • 'mvApi.render(id)'
  • 'mvApi.render(id, modelChange)'
  • 'mvApi.render(id, modelChange, callback)'
  • 'mvApi.render(id, callback)'
  • 'mvApi.render(id, callback, modelChange)'
models.js

The models.js module provides two methods for the 'common.js' module:

  • getData() - which is being invoked if an AJAX request to a service is required. After receiving a successful response, it injects it to the data provided and a new Backbone model created by view in a next step will be based on this joined object.
  • create() - which is being invoked by a view and it simply creates and returns an instance of Backbone model based on data provided.

These methods are private for the whole mvApi scope - there is no need to require this module and invoke its methods directly inside any of the Buildkit modules!

views.js

The views.js module provides one method for the 'common.js' module:

  • render() - which is a wrapper around two other private methods renderItem() and renderCollection() and invokes one of them accordingly to current conditions.

This method is private for the whole mvApi scope - there is no need to require the module and invoke its method directly inside any of the Buildkit modules!
