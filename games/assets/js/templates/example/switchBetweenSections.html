I. Create page layout

In the first place, you'll need to create main page view, which will be a placeholder for other views, loaded later.

  1. Create main page view

    • Create all folders and files in the usual way, as described in Step by step tutorial
    • In the common.js file load 'mvApi' and common models, as described in Step by step tutorial
      These models will be shared between sections
    • Load your SPI to see if everything works correctly before proceeding

  2. Create section view

    • Create new section script similarly to the way you've created the page view
    • In the common.js file load 'your_section_name.js' script and 'your_section_name_models.js' with RequireJS
      These models will be unique for your section
    • The 'your_section_name_models.js' should return, among others, a key 'pageModel' containing values for your main page view, e.g.

      'pageModel': {
          'defaults': {
              'contentHeader': 'Your header'
          }
      }
    • The 'your_section_name.js' script should return a module object (as it usually does all over the Buildkit), containing two public elements:

      'init' {method} which should initialize and load your module layout
      'pageModel' {object} which is the 'pageModel' exposed by 'your_section_name_models.js'

  3. Load your section from main page

    • In your common.js script create a method called e.g. 'renderSection' that will take a section object as an parameter and will use it to provide modelChange and callback for mvApi.render() method as per usual usage described in Step by step tutorial, e.g.
      renderSection = function renderSection(section) {
          mvApi.render('page', section.pageModel, section.init);
      };
    • Invoke renderSection(your_section_name) somewhere in your logic inside common.js
    • Load your SPI to see it in action
